# 🛠️ Локальные утилиты

Космический Кот Чат теперь включает мощные локальные утилиты для улучшения производительности и управления данными.

## 📦 Кэширование (LocalCache)

Система локального кэширования для быстрого доступа к часто используемым данным.

### Возможности:
- Автоматическое истечение срока действия (TTL)
- Ограничение размера кэша
- Автоматическая очистка устаревших записей

### API:
```javascript
// Сохранить в кэш
cache.set('key', data, 300000); // 5 минут

// Получить из кэша
const data = cache.get('key');

// Очистить кэш
cache.clear();
```

### Эндпоинты:
- `GET /api/cache/stats` - статистика кэша
- `POST /api/cache/clear` - очистить кэш

## 💾 Система бэкапов (BackupManager)

Автоматическое создание резервных копий данных чата.

### Возможности:
- Ежедневные автоматические бэкапы
- Хранение последних 7 бэкапов
- Восстановление из бэкапа
- JSON формат для легкого чтения

### Что сохраняется:
- Пользователи (последние записи)
- Сообщения (последние 1000)
- Игры (последние 100)
- Опросы (последние 50)

### Эндпоинты:
- `POST /api/backup/create` - создать бэкап
- `GET /api/backup/list` - список бэкапов

### Файлы бэкапов:
Сохраняются в папке `backups/` с именами вида:
```
backup_2024-01-15_1705123456789.json
```

## 📊 Статистика (ChatStats)

Подробная аналитика активности чата и системы.

### Типы статистики:

#### Дневная статистика:
- Количество сообщений
- Активные пользователи
- Отправленные файлы
- Созданные игры и опросы
- Топ пользователей по активности

#### Недельная статистика:
- Динамика по дням
- Популярные типы игр
- Общие показатели

#### Системная статистика:
- Использование памяти
- Время работы сервера
- Размер базы данных
- Активные сессии

### Эндпоинты:
- `GET /api/stats/daily` - статистика за день
- `GET /api/stats/weekly` - статистика за неделю
- `GET /api/stats/system` - системная статистика

## 📝 Логирование (Logger)

Система логирования событий и ошибок.

### Типы логов:
- `app` - основные события приложения
- `chat` - сообщения чата
- `users` - действия пользователей
- `games` - игровые события
- `system` - системные события

### Возможности:
- Автоматическая ротация логов
- Разделение по типам
- Очистка старых логов
- Структурированный формат

### Использование:
```javascript
logger.info('User connected', { userId: 123 });
logger.error('Database error', error);
logger.chatMessage('username', 'Hello world');
```

### Эндпоинты:
- `GET /api/logs?type=app&days=1` - получить логи

### Файлы логов:
Сохраняются в папке `logs/` с именами вида:
```
app_2024-01-15.log
chat_2024-01-15.log
users_2024-01-15.log
```

## 📁 Утилиты файлов (FileUtils)

Набор функций для работы с файлами.

### Возможности:
- Форматирование размера файлов
- Определение типа файла
- Валидация файлов
- Создание миниатюр (в браузере)
- Сохранение/загрузка JSON файлов

### Функции:
```javascript
FileUtils.formatFileSize(1024); // "1 KB"
FileUtils.isImageFile('photo.jpg'); // true
FileUtils.validateFile(filename, size, type); // {valid: true}
```

## 🔧 Настройка

### Переменные окружения:
```bash
# Основные настройки остаются теми же
DATABASE_URL=postgresql://user:password@localhost:5432/chatdb
PORT=3000
NODE_ENV=production
```

### Автоматические процессы:
- **Бэкапы**: каждые 24 часа
- **Очистка кэша**: каждые 30 минут
- **Очистка логов**: при запуске (старше 7 дней)
- **Очистка сессий**: каждые 10 минут

## 🚀 Преимущества

1. **Независимость**: Нет внешних API зависимостей
2. **Производительность**: Локальный кэш ускоряет работу
3. **Надежность**: Автоматические бэкапы защищают данные
4. **Мониторинг**: Подробная статистика и логи
5. **Простота**: Все утилиты работают "из коробки"

## 🔄 Миграция

При обновлении с версии 1.3.0:
1. Удалите папку `api/` (если есть)
2. Перезапустите сервер
3. Утилиты инициализируются автоматически
4. Первый бэкап создастся через 24 часа

## 🆘 Устранение неполадок

### Проблемы с бэкапами:
- Проверьте права на запись в папку проекта
- Убедитесь в наличии свободного места

### Проблемы с логами:
- Логи создаются автоматически при первом событии
- Проверьте переменную `NODE_ENV` для debug логов

### Проблемы с кэшем:
- Кэш хранится только в памяти
- При перезапуске сервера кэш очищается

Все утилиты работают независимо и не влияют на основную функциональность чата.
